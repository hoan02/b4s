# Baseus Full Companion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn B4S into a tested local companion for supported Baseus earbuds, including first-time BLE discovery/binding, model metadata, state sync, and APK-verified controls.

**Architecture:** Keep Tauri/SolidJS/Rust. Add pure Rust modules for advertisement parsing, connection handshake, and model metadata; keep btleplug I/O behind the existing BLE manager; expose typed state/events to the existing UI. Remote product metadata is optional and never blocks local control.

**Tech Stack:** Rust 2021, btleplug 0.11, Tokio, Tauri 2, SolidJS, TypeScript, Vitest-free Rust unit tests.
